AI Trip Planner
A conversational AI skill that turns scattered travel research into a structured, beautiful HTML travel guide.
Trip Planner Preview

Overview:

Planning a trip usually means juggling dozens of browser tabs — flight searches, hotel comparisons, blog recommendations, Google Maps pins, and scattered notes. I built an AI-powered trip planner that consolidates this entire workflow into a single conversation. The skill asks the right questions, researches real-time prices and logistics, then generates a self-contained HTML travel guide with interactive maps, day-by-day itineraries, and live budget tracking.

My Role:

As both designer and developer, I designed the conversational flow, defined the information architecture for the output, and built the complete system as a Claude Code skill. This included prompt engineering, multi-round data collection logic, real-time web research integration, and the visual design system for the generated HTML guides.

Company & Timeframe:

Personal Project · 2025–2026

01. The Problem
Scattered tabs illustration

Travel planning is fragmented across dozens of tools and tabs. Users bounce between Google Flights, Booking.com, travel blogs, Google Maps, and messaging apps to coordinate with travel partners. Key information gets lost, budgets are vague, and the final "plan" is usually a messy Google Doc or scattered screenshots.

Key Pain Points:

  • Research is scattered across 10+ browser tabs with no single source of truth
  • Budget tracking is manual and often forgotten until overspending occurs
  • Coordinating with travel partners means copying and pasting across apps
  • Logistics details (airport transfers, SIM cards, visa requirements) are afterthoughts
02. Design Approach

The core insight was that travel planning is inherently conversational — people naturally describe trips through dialogue, not forms. The skill's UX was designed around this principle, using structured conversation rounds to progressively build a complete trip profile.

Conversation flow screenshot

Multi-Round Information Gathering

The skill uses structured conversation rounds to collect trip parameters — dates, budget, travel style, dietary restrictions, interests. Each round builds on previous answers, creating a natural dialogue flow rather than a rigid questionnaire.

Smart Defaults with Escape Hatches

Every question offers concrete options based on real destination data, but users can always override. This reduces decision fatigue while preserving full control over the planning process.

Progressive Disclosure

Start with essentials (where, when, budget), then layer in preferences (food, activities, pace). This mirrors how people naturally think about trips — big decisions first, details later.

Research-Backed Suggestions

The skill searches real-time flight prices, hotel rates, and local recommendations before presenting options. Users make decisions based on actual data, not guesswork.

03. What Gets Built

The output is a self-contained HTML file — a complete travel guide that works offline on any device. No app installation, no account creation, no internet required after generation.

Trip guide preview — interactive map + itinerary

Interactive Map

All locations pinned and color-coded by day, with walking routes and proximity grouping for efficient daily plans.

Day-by-Day Itinerary

Time blocks, walking routes, and meal recommendations for each day — structured but flexible enough for spontaneity.

Live Budget Dashboard

Tracks spending across categories (flights, hotels, food, activities) with real-time totals and remaining budget visibility.

Booking Status Tracker

Visual status for flights, hotels, and activities — what's booked, what's pending, and what still needs action.

Packing Checklist

Customized to destination weather and planned activities, with interactive checkboxes for tracking progress.

Digital Nomad Mode

Coworking spots and wifi-friendly cafes surfaced alongside the itinerary for remote workers traveling abroad.

04. Key Design Decisions
1

Conversation over Forms

A chat-based flow feels natural and adapts to each user's situation, unlike a rigid form that assumes everyone plans the same way. Some travelers have fixed dates and open budgets; others have tight budgets and flexible schedules. Conversation handles both.

2

HTML over PDF

Self-contained HTML files are interactive, searchable, and work offline on any device. No app installation needed. Maps are clickable, checklists are interactive, and the budget dashboard updates as users mark items as booked.

3

Budget-First Planning

By establishing budget early and showing real prices, users make informed trade-offs instead of discovering they've overspent after booking. The budget dashboard is central to the guide, not an afterthought.

4

Bilingual by Default

Every guide includes both English and the local language for restaurant names, addresses, and transit directions. This is essential for non-English-speaking destinations — showing a taxi driver an address in the local script makes the difference between arriving and getting lost.
